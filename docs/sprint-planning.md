# Sprint Planning

## Sprint Cadence

- Two-week sprint cycles
- Weekly review with the founder and architecture lead
- Prioritize reliability, security, and product clarity

## Sprint Structure

1. Planning: confirm scope, risks, and dependencies.
2. Execution: implement, review, and test increments.
3. Review: validate outcomes against the roadmap.
4. Retrospective: document improvements and next steps.

## Definition of Done

- Implementation complete
- Tests added or updated
- Documentation updated when needed
- CI green
- Security review considered
